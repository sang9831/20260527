Roadmap

단계별 테크트리

백엔드 취업 준비를 초기, 중기, 실전 단계로 나누어 우선순위를 분명하게 잡습니다.

01

언어 기초

Java 기준 문법, 객체지향, 컬렉션, 예외 처리, 람다와 스트림을 정리합니다.

  • 문법과 제어 흐름
  • 객체지향 설계 원칙
  • 컬렉션과 제네릭
02

CS 기초

네트워크, 운영체제, 자료구조, 알고리즘의 최소 합격선을 확보합니다.

  • HTTP, DNS, TCP/IP
  • 프로세스와 스레드
  • 정렬, 탐색, 해시
03

백엔드 프레임워크

Spring Boot로 REST API, 계층 구조, 의존성 주입, 예외 핸들링을 다룹니다.

  • Controller / Service / Repository
  • Validation / Security
  • 로그와 에러 처리
04

데이터베이스

관계형 모델, 인덱스, 트랜잭션, 정규화, 성능 개선 포인트를 학습합니다.

  • MySQL / PostgreSQL
  • 쿼리 튜닝과 인덱스
  • 트랜잭션과 격리 수준
05

실전 API

인증, 권한, 페이징, 검색, 파일 업로드가 포함된 서비스형 API를 만듭니다.

  • JWT / OAuth 개념
  • DTO 설계
  • RESTful 규칙
06

테스트와 품질

단위 테스트, 통합 테스트, 테스트 코드 작성 습관을 포트폴리오에 반영합니다.

  • JUnit / Mockito
  • API 문서화
  • 리팩터링 기준
07

배포와 운영

Docker, CI/CD, 모니터링, 클라우드 배포 경험으로 실무 감각을 보여줍니다.

  • Docker / Compose
  • GitHub Actions
  • Cloud Deploy
08

면접 대비

프로젝트 설명, 트러블슈팅, 설계 근거를 정리해 질문에 즉답할 수 있게 준비합니다.

  • 회고 문서 작성
  • 기술 선택 이유 정리
  • 면접 예상 질문

Portfolio

포트폴리오에 들어가야 할 것

채용자는 기능보다 문제 해결력을 봅니다. 기능 목록과 설계 이유를 함께 보여주세요.

회원가입, 로그인, 검색, 좋아요, 댓글처럼 기본적인 서비스 흐름을 완성하고, 인증과 예외 처리까지 포함합니다.

캐시 적용, 쿼리 최적화, 비동기 처리, 배치 작업 같은 운영 관점 요소를 넣어 한 단계 더 깊은 설계를 보여줍니다.

Git 브랜치 전략, 코드 리뷰 규칙, Docker 기반 배포, CI 파이프라인을 통해 협업 역량을 설명합니다.

제출 전 체크리스트

  • README에 서비스 목적과 실행 방법이 있는가
  • ERD와 API 문서가 함께 제공되는가
  • 예외 처리와 검증 로직이 정리되어 있는가
  • 테스트 코드가 최소 핵심 로직을 커버하는가
  • 배포 URL과 데모 계정이 명시되어 있는가
Plan

8주 학습 플랜

짧게 집중해서 결과를 만드는 일정입니다. 매주 결과물을 남기는 방식으로 진행하세요.

1-2주

언어와 CS 기초

Java 문법, OOP, HTTP, 네트워크, 자료구조를 정리하고 작은 예제 코드를 만들어 봅니다.

3-4주

Spring Boot와 DB

CRUD API, JPA, 트랜잭션, 예외 처리, Validation을 학습해 기본 서비스 구조를 만듭니다.

5-6주

실전 기능 확장

인증, 검색, 페이징, 캐시, 파일 업로드, 테스트 코드, 문서화를 추가합니다.

7-8주

배포와 지원 준비

Docker와 배포 설정을 정리하고, 이력서와 면접 답변 자료를 최종 보완합니다.
