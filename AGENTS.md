# Agent Guidelines (AGENTS.md)

이 문서는 이 프로젝트에서 작업하는 모든 AI 에이전트가 준수해야 하는 행동 지침, 코딩 표준 및 소통 방식에 대한 정의입니다.

---

## 1. 핵심 원칙 (Core Principles)

1. **상세하고 친절한 설명 제공 (Detailed & Friendly Explanations)**
   - **에이전트 행동 지침:** 사용자의 요청이나 질문에 답할 때, 혹은 코드 변경 작업을 수행한 후에는 단순히 결과를 요약하거나 코드 블록만 제시해서는 안 됩니다.
   - **분량 및 깊이 규정:** 변경 목적, 구체적인 구현 세부사항, 미치는 영향 등을 포함하여 **최소 100자 이상의 상세하고 친절한 설명**을 반드시 덧붙여 전달해야 합니다.
   - **어조:** 전문적이면서도 친절하고 협력적인 톤앤매너를 유지합니다.

2. **디자인 시스템 준수 (Design System Adherence)**
   - [DESIGN.md](file:///c:/coding/AIBE7/20260527/DESIGN.md)에 정의된 **Vibrant Tech** 테마의 색상 팔레트, 타이포그래피(Inter font), 간격 및 둥글기(Shape) 규칙을 엄격히 준수합니다.
   - 임의의 색상 코드나 스타일을 사용하지 않고 CSS 변수나 정의된 테마 스펙에 맞춰 스타일을 설계합니다.

3. **점진적이고 안전한 구현 (Incremental & Safe Implementation)**
   - 코드 변경 시 기존 주석과 문서화 구문을 최대한 유지 및 보존합니다.
   - 변경 사항은 논리적 단위로 분할하여 수행하며, 필요한 경우 `git-commit-convention` 스킬을 활용하여 일관된 커밋 형식을 유지합니다.

---

## 2. 세부 커뮤니케이션 룰 (Detailed Communication Rules)

에이전트는 사용자와 소통 시 다음 가이드라인에 따라 답변을 구성해야 합니다.

* **요청 분석 단계:** 사용자가 요청한 사항을 재해석하여 에이전트 관점에서의 분석 결과를 명확히 공유합니다.
* **설명 요구 조건:**
  ```text
  [필수 준수 사항]
  - 답변 또는 작업 완료 보고 시 본문 텍스트 기준 공백 제외 최소 100자 이상의 풍부한 설명 제공
  - 구현 배경, 기술적 의사 결정의 이유, 주요 코드 변경점에 대한 친절한 설명 포함
  ```
* **결과 검증:** 코드가 정상 작동하는지 검증한 방법과 그 결과를 상세히 기술합니다.

---

## 3. 기술 스택 및 개발 가이드 (Technology Stack & Guide)

* **프론트엔드/백엔드:** HTML, 바닐라 CSS, JavaScript 중심의 구조를 따릅니다.
* **부트스트랩/CSS:** 필요한 경우 `bootstrap` 스킬 가이드라인에 맞춘 반응형 레이아웃 및 컴포넌트 구성을 고려합니다.
* **SEO 최적화:** 웹 애플리케이션 개발 시 적절한 시맨틱 태그 구조 및 메타 데이터를 제공합니다.
